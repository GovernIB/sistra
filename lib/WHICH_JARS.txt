Los jars colocados en esta carpeta se utilizan para compilar los módulos.

Además muchos de estos jars se deben incluir en el servidor de aplicaciones
para que funcione la plataforma. La inclusión de estos jars se realiza a 
través de un ear que carga todos los jars dependientes. Este ear se construye
en el módulo: modul-libs. Para ver los jars incluidos se puede consultar el
fichero: \moduls\modul-libs\etc\application.xml dónde se incluyen los jars 
como java modules.

APACHE CXF: Los jars de Apache CXF no se incluyen en el módulo de librerías comunes
ya que existen jars que dan errores al deployarlos como java modules. Esto es 
debido a un error que da Jboss cuando se intenta deployar un jar que contiene un
xml en su META-INF. La mayoría de los jars que dan problemas pueden "arreglarse"
eliminando el directorio maven de su META-INF, pero el jar cxf-2.2.2.jar tiene xmls 
de configuración necesarios para el funcionamiento de CXF en su META-INF. Por tanto
los jars de CXF se incluyen en cada módulo web que implementa los webservices dentro 
del directorio WEB-INF/lib.
