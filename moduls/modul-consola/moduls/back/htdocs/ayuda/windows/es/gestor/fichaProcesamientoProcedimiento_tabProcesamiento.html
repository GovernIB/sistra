CONFIGURACIÓN DE PROCESAMIENTO

Esta ventana contiene las opciones de configuración para el procesamiento de las solicitudes del procedimiento contra el Backoffice correspondiente.

DESCRIPCIÓN DE CAMPOS

Los campos que aparecen en la ventana son los siguientes:
  • Identificador: Código que identifica unívocamente el procedimento. Este campo es no editable.
  • Descripción: Texto descriptivo del procedimento. Este campo es no editable.
  • Activar procesamiento: Esta casilla permite habilitar el aviso de entradas al Backoffice. Si se marca esta casilla aparecerán el resto de campos que se describen a continuación.
  • Avisos: Apartado que permite configurar las opciones de aviso a Backoffice.
    • Intervalo de avisos (minutos): Este campo permite indicar cada cuantos minutos se avisará al Backoffice que tiene entradas pendients de procesar.
    • Aviso inmediato: Esta casilla permite habilitar/deshabilitar el aviso inmediato de las entradas al Backoffice una vez que lleguen a la Safata Telemática.
  • Tipo aviso: Este apartado permite configurar las opciones relatives a la tecnologia a utilizar para avisar al Backoffice de las entradas a procesar. A continuación se exponen las diferentes opciones:
    • EJB: Indica que la comunicación se llevará a cabo utilizando EJBs. Si se selecciona esta opción se tendrán que cumplimentar las configuraciones siguientes:
      • Localización: Casilla de selección única que permite establecer la localización del EJB que recibirá el aviso de nuevas entradas. Si este EJB se encuentra en el mismo servidor que la plataforma se tendrá que seleccionar Local, en caso contrario Remoto.
      • JNDI: Campo en el que se ha de especificar el JNDI de acceso al EJB.
      • Url: En caso que el EJB esté remoto a SISTRA, este campo ha de contener la URL de acceso al servidor en el que se encuentra el EJB.
    • Webservice: Indica que la comunicación se llevará a cabo utilizando Webservices
      • Url: Este campo ha de contener la URL del endpoint del Webservice.
      • Versión: En este campo se ha de seleccionar la versión del Webservice a utilizar para el aviso de las entradas.
  • Autenticación explícita: Casilla de selección única que permite indicar el tipo de autenticació a utilizar en el aviso de las entradas. Si se selecciona No, no se utilizará autenticación, si se selecciona Usuario/Password se utilizará el usuario y password indicados, y si se selecciona Autenticación organismo se utilizará el usuario y password indicados para el procedimiento en el fichero de propiedades del plugin de login de SISTRA.
