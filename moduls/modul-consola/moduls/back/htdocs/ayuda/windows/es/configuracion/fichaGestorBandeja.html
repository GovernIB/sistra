ALTA / EDICIÓN DE GESTOR

Esta ventana contiene las opciones de configuración del nuevo gestor o del gestor seleccionado para edición.

DESCRIPCIÓN DE CAMPOS

Los campos que aparecen en la ventana son los siguientes:
  • Identificador: Código de usuario del gestor. Este código debe ser el mismo que el utilizado por el usuario para la autenticación.
  • Email: Correo electrónico del gestor. Cualquier aviso email que tenga como destinatario el gestor se enviará a esta dirección de correo electrónico
  • Procedimientos gestionados: Listado de procedimientos gestionados por el usuario. Pulsando el botón Añadir se le mostrará una nueva ventanta con el listado de procedimientos existentes, pudiendo añadir al listado los que considere oportunos. De la misma manera, podrá revocar los permisos del gestor sobre un procedimiento concreto seleccionándolo en el listado y pulsando el botón Quitar
  • Opciones de aviso: Este apartado permite configurar los avisos a recibir por parte del gestor. Estos avisos estarán referidos a los procedimientos que gestiona el usuario. El detalle de los campos es el siguiente:
    • Avisar nuevas entradas: Este check permite habilitar el envio de avisos periódicos al usuario sobre las nuevas entradas recibidas en los procedimientos gestionados. La periodicidad de recepción de los envíos es la misma para todos los gestores y se configura a nivel de propiedades de la aplicación
    • Avisar estado notificaciones: Este check habilita la posibilidad de que los gestores de un procedimiento reciban avisos sobre las notificaciones asociadas al mismo en el mismo mail de aviso de entradas
    • Aviso de monitorización: Este check habilita el envío al gestor de un correo electrónico de aviso que informa del estado de las solicitudes recibidas en los procedimientos de los que el usuario es gestor. La periocidad de envío de estos avisos es la misma para todos los gestores y se configura a nivel de propiedades de la aplicación
  • Opciones aplicación bandeja telemática: Este apartado permite configurar opciones relativas a la Bandeja Telemática para el gestor. El detalle de los campos es el siguiente:
    • Permitir cambio de estado entradas Este check permite habilitar al gestor, en la Bandeja Telemática, para que modifique individualmente el estado de las entradas asociadas a los procedimientos gestionados
    • Permitir cambio masivo estado entradas Este check permite habilitar al gestor, en la Bandeja Telemática, para que modifique de manera masiva el estado de las entradas asociadas a los procedimientos gestionados
    • Permitir gestión manual expedientes Este check permite habilitar el acceso del gestor al Módulo Manual de Gestión de Expendientes de la Bandeja Telemática. Desde este módulo el gestor podrá crear expedientes, notificaciones y/o comunicaciones asociadas a sus procedimientos
