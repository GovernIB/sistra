CONFIGURACIÓN DE DOMINIO

Esta ventana contiene las opciones de configuración del nuevo dominio o del dominio seleccionado para edición.

DESCRIPCIÓN DE CAMPOS

Los campos que aparecen en la ventana son los siguientes:
  • Identificador: Código que identifica unívocamente el dominio. Este código debe ser un alfanumérico de 10 posiciones y es recomendable que sea un código nemotécnico
  • Descripción: Texto descriptivo del dominio. Este texto debe estar formado por un máximo de 100 carácteres alfanuméricos que describan el dominio en cuestión
  • Configuración de cacheo:
  • Tipo: Este apartado permite establecer el tipo del dominio y las configuraciones asociadas. El detalle de cada campo es el siguiente:
    • SQL: Esta casilla indica que el dominio es de tipo SQL. Este tipo de dominio consiste en la ejecución de una consulta SQL sobre un datasource especificado. Si se marca esta casilla, tendrá que rellenar los datos siguientes:
      • Datasource: Ruta JNDI del Datasource de conexión con la BBDD en la que se ejecutará la consulta
      • Query: Sentencia SQL que resuelve el dominio. Los parámetros del dominio se deberán especificar con un interrogante ("?")
    • EJB: Esta casilla indica que el dominio es de tipo EJB. Este tipo de dominio accederá a una aplicación que cumpla con el API de integración
      • Localización: Indica la localización del EJB respecto al servidor en el que está desplegada la plataforma de tramitación. La localización puede ser local, si se encuentra en el mismo servidor, o remoto si se encuentra en un servidor diferente.
      • JNDI: Indica el JNDI de acceso al EJB
      • Url: En caso de localización remota, indica la url del servidor en el que está ubicado el EJB. Este campo no es necesario en caso de localización local
    • Webservice: Esta casilla indica que el dominio es de tipo WS. Este tipo de dominio accederá a una aplicación que cumpla con el API de integración
      • Url: Url del Webservice.
      • Versión: Indica la versión del Webservice a utilizar. Las versiones de los WS y sus funcionalidades están especificadas en la documentación de integración de la Forja
      • Soap Action: Permite indicar un Soap Action específico a utilizar en la llamada al dominio
    • Fuente de datos: Esta casilla indica que el dominio es de tipo fuente de datos. Este tipo de dominios acceden a un conjunto de datos mantenidos por el gestor a través de la opción fuente de datos de Bantelfront
      • Pseudo-Query: En este campo se indica la sentencia pseudoSQL para la consulta de la fuente de datos. El formato de la consulta debe cumplir lo siguiente

        • SELECT campo (,campo)
        FROM fuenteDatos
        WHERE expresion ( [AND | OR] expresion ] )*
        ORDER BY campo

        donde expresion: campo [= | LIKE] ?, siendo "?" un parámetro.

        Las partículas de la sentencia pseudoSQL(SELECT, FROM, WHERE, AND, OR, LIKE, ORDER BY) deben estar en mayúsculas.
        Respecto a las comparaciones en el WHERE serán ignorecase (= y LIKE).
        Respecto al LIKE se realizará automáticamente la comparación: %parametro%.
        Respecto al ORDER BY sólo se puede indicar un campo. Además como todos los campos son de tipo cadena, la ordenación es literal por lo que si se quiere crear un campo en la fuente de datos que permita ordenar secuencialmente, es conveniente usar letras en lugar de números.

  • Probar dominio: Si clica el botón asociado a este campo, se le abrirá una ventana en la que podrá realizar una prueba de invocación del dominio, indicando previamente el valor de los parámetros correspondientes.
