Descripció
Identificador Identificador del domini
Tipus Indica com es resol el domini: EJB, Webservice, consulta directa a BBDD o consulta a font de dades.
Un domini de tipus EJB o Webservice accedirà a una aplicació que compleixi amb el api d'integració.
Un domini de tipus SQL consistirà a executar una consulta SQL sobre un datasource especificat.
Un domini de tipus font de dades consistirà a accedir a un conjunt de dades mantingudes pel gestor a través de Bantelfront.
Cacheable Indica si la consulta al domini ha de ser cacheada entre diferents invocacions. S'haurà de marcar com cacheables els dominis que es prenguin com constants (províncies, municipis, taules de tipus, etc). Es forçarà el seu refresc una vegada al dia.
URL Según el tipus de domini indicarà:
- Domini EJB: url del servidor on esta ubicat el EJB (en cas de ser remot)
- Dominio Webservice: url del Web Service
- Dominio SQL: path JNDI de Datasource
- Dominio Font de dades: no aplica
Al formar la url es pot emprar la variable @backoffice.url@, que serà reemplaçada segons el valor indicat en els fitxers de configuració.
SQL Per a tipus SQL s'indicarà la sentència SQL que resol el domini. Els paràmetres de la consulta seràn especificats amb "?".

Per a tipus Font de dades s'indicará una sentencia pseudoSQL per consultar una font de dades. El format d'aquesta consulta ha de ser:

SELECT campo (, campo)*
FROM fuenteDatos
WHERE expresion ( [AND | OR] expresion ] )*
ORDER BY campo
on expresion: campo [= | LIKE] ?

Respecte al ORDER BY només es pot indicar un camp. A més com tots els camps són de tipus cadena l'ordenació és literal pel que si es vol crear un camp en la font de dades que permeti ordenar seqüencialment, millor usar lletres que nombres.
Nombre JNDI EJB Per a tipus EJB indicarà el JNDI de accés
Localització EJB Per a tipus EJB indica si el EJB es troba en un servidor remot a la plataforma de tramitació. En cas de ser remot haurá d'indicar-se la url del servidor remot.
Autenticació explícita Per a tipus EJB/Webservice indica si s'ha de realitzar una autenticació explícita (accés amb usuari diferent al que ha iniciat la sessió de tramitació):
  • No: el contenidor d'EJB serà l'encarregat de traslladar l'autenticació (no vàlid per a webservices)
  • Amb usuari/password: s'utilitza l'usuari/password indicat per a autenticar la cridada
  • Autenticació organisme : s'utilitza l'usuari/password suministrat pel plugin d'autenticació explíta de l'organisme per a autenticar la cridada
Usuari/Password
Per a tipus EJB/Webservice en cas de requerir una autenticació explícita s'indica l'usuari/password amb la qual es realitza
