Campo
Descripción
Identificador Identificador del dominio
Cacheable Indica si la consulta al dominio debe cachearse entre diferentes llamadas. Se deberá marcar como cacheables los dominios que se tomen como constantes (provincias, municipios, tablas de tipos, etc). Se forzará su refresco una vez al día.
Tipo Indica como se resuelve el dominio: EJB, Web Service, consulta directa a BBDD ó consulta a fuente de datos.
Un dominio de tipo EJB ó Webservice accederá a una aplicación que cumpla con el api de integración.
Un dominio de tipo SQL consistirá en ejecutar una consulta SQL sobre un datasource especificado.
Un dominio de tipo fuente de datos consistirá en acceder a un conjunto de datos mantenidos por el gestor a través de Bantelfront.
URL Según el tipo de dominio indicará:
- Dominio EJB: url del servidor dónde esta ubicado EJB. Si es local no hace falta indicar la url, sólo es necesario en caso de ser remoto.
- Dominio Webservice: url del Web Service
- Dominio SQL: path JNDI de Datasource
- Dominio Fuente datos: no se usa
Al formar la url se puede emplear la variable @backoffice.url@, que será sustituida por según el valor indicado en los ficheros de configuración
SQL
  • Para tipo SQL se indicará la sentencia SQL que resuelve el dominio. Los parámetros de la consulta serán especificados con "?".

  • Para tipo Fuente de datos se indicará una sentencia pseudoSQL para consultar una fuente de datos. El formato de esta consulta debe ser el siguiente:

    SELECT campo (, campo)*
    FROM fuenteDatos
    WHERE expresion ( [AND | OR] expresion ] )*
    ORDER BY campo

    donde expresion: campo [= | LIKE] ?, siendo "?" un parámetro.

    Las partículas de la sentencia pseudoSQL (SELECT, FROM, WHERE, AND, OR, LIKE, ORDER BY) deben estar en mayúsculas.
    Respecto a las comparaciones en el WHERE serán ignorecase ( = y LIKE ).
    Respecto al LIKE se realizará automáticamente la comparación: %parametro% .
    Respecto al ORDER BY sólo se puede indicar un campo. Además como todos los campos son de tipo cadena la ordenación es literal por lo que si se quiere crear un campo en la fuente de datos que permita ordenar secuencialmente, mejor usar letras que números.
Nombre JNDI EJB Para tipo EJB indicará el JNDI de acceso
Localización EJB Para tipo EJB indicará si el EJB se encuentra en un servidor remoto a la plataforma de tramitación. En caso de ser remoto deberá indicarse la url del servidor remoto.
Autenticación explicita Para tipo EJB/Webservice indica si se debe realizar una autenticación explícita (acceso con usuario diferente al que ha iniciado la sesión de tramitación):
  • No: el contenedor de EJB será el encargado de trasladar la autenticación (no válido para webservices)
  • Con usuario/password: se utiliza el usuario/password indicado para autenticar la llamada
  • Autenticación organismo : se utiliza el usuario/password suministrado por el plugin de autenticación explíta del organismo para autenticar la llamada
Usuario / Password Para tipo EJB/Webservice en caso de requerir una autenticación explícita se indica el usuario/password con la que se realiza
