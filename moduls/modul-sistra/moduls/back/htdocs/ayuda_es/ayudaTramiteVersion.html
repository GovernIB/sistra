Para poder modificar una versión de un trámite previamente hay que Bloquear la versión.

Campo
Descripción
Versión Versión del trámite
Motivo Motivo que genera el cambio de versión para control interno.
Tag cuaderno de carga Tag que permite verificar al desarrollador la versión puesta en producción.
Fecha exportación trámite Fecha en la que se exportó el xml del trámite con el cual ha sido importado. Sirve para identificar que xml ha sido utilizado para importar un trámite.
Plazo presentación En caso de que el trámite disponga de plazo de presentación se puede especificar dicho plazo
Idiomas Se indica la lista de idiomas soportados para el trámite. Al modificar cualquier elemento de la versión que requiera traducción se solicitará que se introduzca la traducción para estos idiomas.
Debe firmarse Indica que el trámite debe firmarse en el paso de Enviar. Si el usuario no dispone de certificado digital el tipo de tramitación será presencial. El trámite deberá firmarse por el representante.
En caso de que se acceda en forma delegada deberá firmarse por un delegado que tenga permiso para enviar trámites en nombre de la entidad.
Circuito reducido Para trámites que sólo tengan un formulario y no requieran firma puede habilitarse el esquema reducido que reduce el número de pantallas.
Ir directo a url fin Para trámites con circuito reducido pasa a la url de finalización sin pasar por el paso de justificante
Autenticación anónima por defecto Si el trámite tiene activado el nivel anónimo y otros niveles se puede indicar que si se accede al trámite y no se esta autenticado, se autentique automáticamente por defecto de forma anónima
Registro automnático Al acceder al paso de registrar se realizará automáticamente el envnío.
Destino del trámite Indica destino del trámite:
- Registro: pasa por registro de entrada (y posteriormente a la bandeja)
- Bandeja: va directo a la bandeja (sin pasar por registro de entrada). Para destino Bandeja se puede habilitar la confirmación automática: los preenvios se confirman automáticamente tras su envío (es decir, no necesitan ser confirmados en el punto de registro) y pasan a la Bandeja.
- Consulta: trámites que realizan una consulta y obtengan como resultado uno o más documentos. (estos trámites no realizan entrada en bandeja y sólo permiten tener formularios)
- Asistent: tràmits que utilitzen la plataforma per a rellenar els formularis per després presentarlos presencialment. (aquests tràmits no realitzen entrada en safata i només permeten tenir formularis)
Organo destino Organo al que va dirigido el trámite. La lista de servicios destino es proporcionada por el registro del organismo.
Unidad administrativa Unidad Administrativa responsable trámite. La lista de unidades administrativas se establece mediante los dominios que establecen la estructura orgánica del organismo (GESACUNADM y GESACARBUA)
Oficina Registro Oficina de registro dónde se realizará el apunte registral. La lista de oficinas de registro es proporcionada por el registro del organismo. Para trámites que no pasan por registro puede tomar cualquier valor.
Tipo Asunto Tipo de asunto del apunte registral. La lista de oficinas de registro es proporcionada por el registro del organismo. Para trámites que no pasan por registro puede tomar cualquier valor.
Tipo de acceso Para un trámite de tipo consulta indica si se accede al Backoffice mediante EJB o Webservice
Url acceso Indica la url de acceso (necesaria en caso de ser webservice o EJB remoto)
Al formar la url se puede emplear la variable @backoffice.url@, que será sustituida por según el valor indicado en los ficheros de configuración
EJB de Consulta Cuando el tipo de acceso es EJB indica el JNDI de un EJB que implemente el interfaz de destino de consulta que devuelva los documentos de respuesta.
Localitzación EJB Indica si el EJB es local o remoto (ubicado en diferente servidor de aplicaciones)
Autenticación explícita Indica si para acceder al EJB/Webservice se debe realizar una autenticación explícita (acceso con usuario diferente al que ha iniciado la sesión de tramitación):
  • No: el contenedor de EJB será el encargado de trasladar la autenticación (no válido para webservices)
  • Con usuario/password: se utiliza el usuario/password indicado para autenticar la llamada
  • Autenticación organismo : se utiliza el usuario/password suministrado por el plugin de autenticación explíta del organismo para autenticar la llamada
Usuario/Password En caso de requerir una autenticación explícita se indica el usuario/password con la que se realiza
