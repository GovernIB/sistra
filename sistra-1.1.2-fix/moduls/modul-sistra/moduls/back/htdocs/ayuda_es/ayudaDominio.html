Campo
Descripción
Identificador Identificador del dominio
Cacheable Indica si la consulta al dominio debe cachearse entre diferentes llamadas. Se deberá marcar como cacheables los dominios que se tomen como constantes (provincias, municipios, tablas de tipos, etc). Se forzará su refresco una vez al día.
Tipo Indica como se resuelve el dominio: EJB, Web Service o consulta directa a BBDD.
URL Según el tipo de dominio indicará:
- url del servidor dónde esta ubicado EJB. Si es local no hace falta indicar la url, sólo es necesario en caso de ser remoto.
- url del Web Service
- path JNDI de Datasource
Al formar la url se puede emplear la variable @backoffice.url@, que será sustituida por según el valor indicado en los ficheros de configuración
SQL Para tipo SQL se indicará la sentencia SQL que resuelve el dominio. Los parámetros de la consulta serán especificados con "?".
Nombre JNDI EJB Para tipo EJB indicará el JNDI de acceso
Localización EJB Para tipo EJB indicará si el EJB se encuentra en un servidor remoto a la plataforma de tramitación. En caso de ser remoto deberá indicarse la url del servidor remoto.
Autenticación explicita Indica si para acceder al EJB/Webservice se debe realizar una autenticación explícita (acceso con usuario diferente al que ha iniciado la sesión de tramitación):
  • No: el contenedor de EJB será el encargado de trasladar la autenticación (no válido para webservices)
  • Con usuario/password: se utiliza el usuario/password indicado para autenticar la llamada
  • Autenticación organismo : se utiliza el usuario/password suministrado por el plugin de autenticación explíta del organismo para autenticar la llamada
Usuario / Password En caso de requerir una autenticación explícita se indica el usuario/password con la que se realiza
