Camp
Descripció
Nivell autenticació Indica els nivells als que s'aplicacará aquestes especificacions.
Un document se li requerirà a l'usuari segons el nivell d'autenticació amb el que l'usuari inicie sessió i els nivells que s'hajen especificat per al document.
Versió Indica la versió del document. Tot document ha d'estar definit en el RDS mitjantçant un model / versió.
Obligatori Indica si el document serà obligatori, opcional o dependrà de dades introduïdes en formularis.
Script Obligatorietat Si el document és depenent, la seua dependència s'haurà d'indicar mitjantçant aquest script. El tipus d'aquest camp és script i espera com resultat: S, N o D.
Script flux tramitació
Indica usuari que ha d'aportar aquest document (sol per a autenticat i si aquesta habilitat el flux de tramitació). Si és nul o retorna buit ha d'aportar-lo qui inicia el tràmit
Signar digitalmente Indica si el document ha de ser signat per l'usuari després d'emplenar-lo (formularis) o adjuntar-lo (annexe). El signant deurà dispondre de certificat digital.
Camp DNI Signant Indica qui haurà de signar el document. Aquest camp és de tipusscript. En cas de no indicar l'script el signant serà qui inicia el tràmit.
Content type firma Indica el content type del documento a firmar. Si no se especifica se utilizará uno por defecto.
Gestor de formularis

Indica el gestor de formularis que es cridará (FORMS,etc)
Model - Versió

Identifica en el gestor de formularis el model/versió del formulari. Per FORMS hi ha un botó de selecció.
Script càrrega de dades inicials

Per a formularis permet precarregar dades en el formulari (p.ex. nif i nom de l'usuari). El tipus d'aquest camp és script i no espera cap resultat.

La forma de precarregar aquestes dades des del script és mitjançant el plugin DATOSINICIALES:
DATOSINICIALES.setDatoInicial('campo1','valor1');


NOTA: Actualment no es pot precarregar camps indexats (llistes, etc.)

Script configuració de formularis

Per a formularis permet:

  • - bloquejar camps en un formulari (p.ex. si precarreguem nif del certificat no permitim editar-lo).
  • - establir propiedats específiques per al gestor de formularis. Per a FORMS disposem de las següents propietats:
    * pantallaFin.mostrar: (true/false) indica si es mostra la darrera pantalla de FORMS
    * pantallaFin.titulo: permet establir el títol de la darrera pantalla de FORMS
    * pantallaFin.texto: permet establir el texte de la darrera pantalla de FORMS

El tipus d'aquest camp és script i no espera cap resultat.

La forma de precarregar aquesta configuració és mitjantçant el plugin CONFIGURACIONFORMULARIO:
CONFIGURACIONFORMULARIO.setDatoSoloLectura('campo1');
CONFIGURACIONFORMULARIO.setPropiedad("pantallaFin.mostrar","true"); CONFIGURACIONFORMULARIO.setPropiedad("pantallaFin.titulo",PLUGIN_MENSAJES.getMensaje("tituloFORMS")); CONFIGURACIONFORMULARIO.setPropiedad("pantallaFin.texto",PLUGIN_MENSAJES.getMensaje("textoFORMS"));

Script de validació de tornada de FORMS Per a formularis permet realitzar una validació sobre les dades del formulari tornat per FORMS. El tipus d'aquest camp és script i espera com resultat S en cas correcte i N en cas d'error.
Script de modificació de tornada de FORMS Per a formularis permet modificar dades d'altres formularis després de tornar de FORMS. Per a això es permet en el plugin de formularis accedir a un mètod per establir dades en altres formularis:
PLUGIN_FORMULARIOS.setDatoFormulario('FOR1',1,'campo1','valorCampo');
Script Plantilla Per a formularis permet establir la plantilla que s'haurà d'utilitzar per a la seua visualització. El tipus d'aquest camp és script i espera com resultat el codi de la plantilla. Si el script retorna cadena buida ('') no s'utilitzará una plantilla especifica i s'utilitzarà la per defecte.
Script per a càlcul dades pagament

Per a pagaments permet realitzar el càlculo de la taxa a pagar. El tipus d'aquest camp és script i no espera cap resultat.

La forma d'indicar les dades del pagament és mitjantçant el plugin DATOSPAGO.

Les dades obligatòries són::

  • Modelo: model de tasa (per defecte 046)
  • Id Tasa: identificador de la tasa
  • Concepto: concepte de la tasa (per defecte el títol del pagament)
  • Fecha devengo: data de cálcul de la tasa en format YYYYMMMDD (per defecte la data actual)
  • Importe: import de la tasa en format cents (1 euro = 100 cents)
  • Nif: nif del declarant (per defecte el nif del representant)
  • Nombre: nom i llinatges del declarant (per defecte nom i llinatges del representant)

Els mètods disponibles son:
DATOSPAGO.setModelo('Modelo');
DATOSPAGO.setIdTasa('IdTasa');
DATOSPAGO.setConcepto('Concepto del pago');
DATOSPAGO.setFechaDevengo('Fecha');
DATOSPAGO.setImporte('Importe');
DATOSPAGO.setNif('Nif');
DATOSPAGO.setNombre('Nombre');

A més es dispondrà d'una funció per a consultar l'importe d'una taxa (en cents):
res = DATOSPAGO.consultarImporteTasa('idTasa'); // En cas d'error torna -1

En cas que es necessitin passar paràmetres a la funció de consultar l'import es pot utilitzar la funció: DATOSPAGO.establecerParametroCalculoTasa('nombreParametro','valorParametro');

D'aquesta forma es poden passar els paràmetres necessaris per a realitzar el càlcul. L'ús de paràmetres en el càlcul de la taxa dependrà del plugin de pagament de l'organisme.

Exemple complet:
res = DATOSPAGO.consultarImporteTasa('1.2.6');
if (res == -1) {
ERRORSCRIPT.setExisteError(true);
}else{
DATOSPAGO.setIdTasa('1.2.6');
DATOSPAGO.setImporte(res);
}
Tipus de pagament Indica el tipus de pagament permés: Telemàtic / Presencial / Ambdós
Informació adicional Permet sobreescriure per nivell d'autenticació la informació específica del document.
